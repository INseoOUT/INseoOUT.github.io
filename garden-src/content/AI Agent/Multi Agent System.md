# Multi Agent System이란
> 🌱 채워가는 중 — fairy를 단일 에이전트에서 멀티 에이전트 계층 구조로 재설계했던 경험이 출발점

하나의 큰 에이전트 대신 역할이 나뉜 여러 에이전트를 엮어 문제를 푸는 구조.

## 학습 목표
- [ ] 계층 구조(라우터 → 자식) vs 평면 협업 구조의 트레이드오프
- [ ] 라우팅 설계 — 판단은 라우터가, 실행은 자식이
- [ ] 에이전트 간 상태 공유의 경계 (무엇을 넘기고 무엇을 감추나)
- [ ] fairy 재설계에서 배운 것 정리 (수집 시간 -60%가 나온 이유)
